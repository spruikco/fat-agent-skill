# CI/CD Integration Guide

Run FAT Agent checks automatically on every deploy. Fail builds that don't meet
quality thresholds, generate badges, and post audit summaries to PRs.

## GitHub Actions (Recommended)

### Full Workflow: `.github/workflows/fat-audit.yml`

```yaml
name: FAT Audit

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  schedule:
    # Weekly audit every Monday at 9am UTC
    - cron: '0 9 * * 1'

jobs:
  fat-audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Fetch live page HTML
        run: |
          curl -sL -o page.html "${{ vars.SITE_URL }}"
          echo "Fetched $(wc -c < page.html) bytes"

      - name: Run FAT audit
        id: audit
        run: |
          python scripts/analyse-html.py --url "${{ vars.SITE_URL }}" page.html \
            | python scripts/calculate-score.py > scores.json

          # Extract overall score and grade
          SCORE=$(python -c "import json; d=json.load(open('scores.json')); print(d['overall']['score'])")
          GRADE=$(python -c "import json; d=json.load(open('scores.json')); print(d['overall']['grade'])")
          echo "score=$SCORE" >> $GITHUB_OUTPUT
          echo "grade=$GRADE" >> $GITHUB_OUTPUT

      - name: Check score threshold
        run: |
          SCORE=${{ steps.audit.outputs.score }}
          THRESHOLD=${{ vars.FAT_THRESHOLD || '60' }}
          if [ "$SCORE" -lt "$THRESHOLD" ]; then
            echo "::error::FAT Score $SCORE is below threshold $THRESHOLD"
            exit 1
          fi
          echo "FAT Score $SCORE meets threshold $THRESHOLD"

      - name: Generate badge
        if: always()
        run: |
          python scripts/generate-badge.py scores.json \
            --image --output fat-badge.svg

      - name: Upload badge artifact
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: fat-badge
          path: fat-badge.svg

      - name: Save to history
        if: github.ref == 'refs/heads/main'
        run: |
          python scripts/track-history.py --save scores.json \
            --url "${{ vars.SITE_URL }}"

      - name: Comment on PR
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const scores = JSON.parse(fs.readFileSync('scores.json', 'utf8'));
            const o = scores.overall;
            const summary = scores.summary || {};
            const critical = (summary.critical || []).length;
            const high = (summary.high || []).length;
            const medium = (summary.medium || []).length;
            const low = (summary.low || []).length;

            const body = `## FAT Audit Report

            | Metric | Score |
            |--------|-------|
            | **Overall** | **${o.grade} ${o.score}/100** |
            | SEO | ${scores.seo.score}/100 |
            | Security | ${scores.security.score}/100 |
            | Accessibility | ${scores.accessibility.score}/100 |
            | Performance | ${scores.performance.score}/100 |

            **Issues:** ${critical} Critical, ${high} High, ${medium} Medium, ${low} Low

            _Generated by [FAT Agent](https://github.com/spruikco/fat-agent-skill)_`;

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: body
            });
```

### Repository Variables

Set these in your repo's Settings > Variables:

| Variable | Example | Description |
|----------|---------|-------------|
| `SITE_URL` | `https://mysite.com` | URL to audit |
| `FAT_THRESHOLD` | `60` | Minimum passing score (default: 60) |

## Netlify

### Build Plugin Approach

Add to `netlify.toml`:

```toml
[build]
  publish = "dist"
  command = "npm run build"

[[plugins]]
  package = "./plugins/fat-audit"

[build.environment]
  FAT_THRESHOLD = "60"
  SITE_URL = "https://mysite.com"
```

Create `plugins/fat-audit/index.js`:

```javascript
module.exports = {
  async onPostBuild({ utils, constants }) {
    const { execSync } = require('child_process');
    const fs = require('fs');
    const threshold = process.env.FAT_THRESHOLD || 60;
    const url = process.env.SITE_URL || process.env.URL;

    try {
      // Fetch the live page
      execSync(`curl -sL -o /tmp/page.html "${url}"`);

      // Run FAT audit
      execSync(
        `python3 scripts/analyse-html.py --url "${url}" /tmp/page.html ` +
        `| python3 scripts/calculate-score.py > /tmp/scores.json`
      );

      const scores = JSON.parse(fs.readFileSync('/tmp/scores.json', 'utf8'));
      const score = scores.overall.score;
      const grade = scores.overall.grade;

      console.log(`FAT Score: ${grade} ${score}/100`);

      if (score < threshold) {
        utils.build.failBuild(
          `FAT Score ${score} is below threshold ${threshold}`
        );
      }
    } catch (error) {
      utils.build.failPlugin(`FAT audit failed: ${error.message}`);
    }
  }
};
```

## Vercel

### Post-Deploy via GitHub Actions

Trigger on Vercel deployment status:

```yaml
name: FAT Audit (Vercel)

on:
  deployment_status:

jobs:
  fat-audit:
    if: github.event.deployment_status.state == 'success'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Run FAT audit on deploy URL
        run: |
          DEPLOY_URL="${{ github.event.deployment_status.target_url }}"
          curl -sL -o page.html "$DEPLOY_URL"
          python scripts/analyse-html.py --url "$DEPLOY_URL" page.html \
            | python scripts/calculate-score.py > scores.json

          SCORE=$(python -c "import json; print(json.load(open('scores.json'))['overall']['score'])")
          echo "FAT Score: $SCORE"
          [ "$SCORE" -ge 60 ] || exit 1
```

## GitLab CI

### `.gitlab-ci.yml`

```yaml
stages:
  - build
  - deploy
  - audit

fat-audit:
  stage: audit
  image: python:3.11-slim
  needs: [deploy]
  script:
    - curl -sL -o page.html "$SITE_URL"
    - python scripts/analyse-html.py --url "$SITE_URL" page.html
        | python scripts/calculate-score.py > scores.json
    - |
      SCORE=$(python -c "import json; print(json.load(open('scores.json'))['overall']['score'])")
      GRADE=$(python -c "import json; print(json.load(open('scores.json'))['overall']['grade'])")
      echo "FAT Score: $GRADE $SCORE/100"
      if [ "$SCORE" -lt "${FAT_THRESHOLD:-60}" ]; then
        echo "Score below threshold!"
        exit 1
      fi
  artifacts:
    paths:
      - scores.json
    when: always
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
  variables:
    SITE_URL: "https://mysite.com"
    FAT_THRESHOLD: "60"
```

To add MR comments, use the GitLab API:

```yaml
  after_script:
```

## Generic Shell Script

Works with any CI system (CircleCI, Bitbucket Pipelines, Jenkins, etc.):

### `fat-audit.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

# Configuration
URL="${SITE_URL:?Set SITE_URL environment variable}"
THRESHOLD="${FAT_THRESHOLD:-60}"
SCRIPTS_DIR="${SCRIPTS_DIR:-scripts}"

echo "=== FAT Audit ==="
echo "URL: $URL"
echo "Threshold: $THRESHOLD"

# Fetch page
curl -sL -o /tmp/page.html "$URL"
echo "Fetched $(wc -c < /tmp/page.html) bytes"

# Run audit pipeline
python3 "$SCRIPTS_DIR/analyse-html.py" --url "$URL" /tmp/page.html \
  | python3 "$SCRIPTS_DIR/calculate-score.py" > /tmp/scores.json

# Extract results
SCORE=$(python3 -c "import json; print(json.load(open('/tmp/scores.json'))['overall']['score'])")
GRADE=$(python3 -c "import json; print(json.load(open('/tmp/scores.json'))['overall']['grade'])")

echo ""
echo "FAT Score: $GRADE $SCORE/100"
echo ""

# Show category breakdown
python3 -c "
import json
d = json.load(open('/tmp/scores.json'))
print(f\"  SEO:           {d['seo']['score']}/100\")
print(f\"  Security:      {d['security']['score']}/100\")
print(f\"  Accessibility: {d['accessibility']['score']}/100\")
print(f\"  Performance:   {d['performance']['score']}/100\")
"

# Check threshold
if [ "$SCORE" -lt "$THRESHOLD" ]; then
    echo ""
    echo "FAIL: Score $SCORE is below threshold $THRESHOLD"
    exit 1
fi

echo ""
echo "PASS: Score $SCORE meets threshold $THRESHOLD"
exit 0
```

### Usage in various CI systems:

**CircleCI** (`.circleci/config.yml`):
```yaml

```

## Tips

- **Set realistic thresholds** — Start at 50-60 and increase as you fix issues
- **Use schedule-based audits** — Weekly cron catches regressions early
- **Track history** — Run `track-history.py --save` after each audit
- **Badge in README** — Generate and commit `fat-badge.svg` for visibility
- **Separate concerns** — Run the audit after deploy, not during build
- **Cache Python** — Use `actions/setup-python` with cache for faster runs
